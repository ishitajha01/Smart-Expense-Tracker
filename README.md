# рџ’° Smart Expense Tracker

A full-stack personal finance web application built with **FastAPI**, **PostgreSQL**, and **vanilla JavaScript**. Features real-time AI-powered expense validation, JWT authentication, automated recurring expenses, budget tracking, savings goals, and CSV/PDF export вЂ” all served from a single deployable backend.

---

## Features

- **Authentication** вЂ” Secure signup and login with bcrypt-hashed passwords and JWT tokens
- **Expense Management** вЂ” Add, edit, soft-delete, and restore expenses with full history tracking
- **AI Validation** вЂ” Google Gemini validates whether a description matches the chosen category before saving
- **AI Insights** вЂ” Personalised monthly spending insights and end-of-month forecast generated by Gemini
- **Budget Tracking** вЂ” Set per-category monthly budgets with rollover support and visual progress alerts
- **Savings Goals** вЂ” Create goals with target amounts, deadlines, and progress tracking
- **Recurring Expenses** вЂ” Mark expenses as daily, weekly, or monthly вЂ” auto-generated on startup
- **Tags** вЂ” Tag expenses for flexible organisation beyond categories
- **Trash & Restore** вЂ” Deleted expenses go to trash and can be restored at any time
- **Reports** вЂ” Export expenses to CSV or PDF, with a yearly summary breakdown
- **Search & Filter** вЂ” Filter by month, category, tags, or search by description
- **Security** вЂ” XSS sanitisation, future date blocking, amount limits, delete confirmation modals

## Local Setup

### Prerequisites
- Python 3.10+
- PostgreSQL running locally
- A Gemini API key from [aistudio.google.com](https://aistudio.google.com)

### Steps

**1. Clone the repo**
```bash
git clone https://github.com/yourusername/SmartExpenseTracker.git
cd SmartExpenseTracker
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Set up environment variables**
```bash
cp .env.example .env
```

Edit `.env`:
```
DATABASE_URL=postgresql://postgres:yourpassword@localhost:5432/expense_tracker
JWT_SECRET=your-long-random-secret-key
JWT_ALGORITHM=HS256
JWT_EXPIRE_DAYS=7
GEMINI_API_KEY=your-gemini-api-key
```

**4. Create the database**
```bash
psql -U postgres -c "CREATE DATABASE expense_tracker;"
```

**5. Run the app**
```bash
uvicorn main:app --reload
```

Open [http://localhost:8000](http://localhost:8000) вЂ” tables are created automatically on first run.

## Deployment on Render

1. Push the project to a GitHub repository

2. Go to [render.com](https://render.com) в†’ **New в†’ Web Service**
   - Connect your GitHub repo
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`

3. Create a **PostgreSQL** database on Render (free tier)
   - Copy the **Internal Database URL**
   - Add it as `DATABASE_URL` in your web service's environment variables

4. Add the remaining environment variables in the Render dashboard:
   - `JWT_SECRET`
   - `GEMINI_API_KEY`

5. Hit **Deploy** вЂ” Render auto-deploys on every push to main

## Security Notes

- Passwords are hashed with **bcrypt** вЂ” never stored in plain text
- All routes are protected with **JWT Bearer token** authentication
- User data is fully isolated вЂ” no user can access another user's data
- Descriptions are **HTML-sanitised** before storage to prevent XSS
- Amount inputs are validated server-side with a max cap of в‚№1,00,00,000
- Future dates are rejected at the schema validation layer
- **Keep your `.env` file out of version control** вЂ” it is listed in `.gitignore`
